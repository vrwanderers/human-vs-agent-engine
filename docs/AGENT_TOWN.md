# Agent 小镇

`agent_town` 是用于观察当前认知架构的生活模拟沙盒。它不是把多个 NPC 写入同一个提示词：`human_agent_coop` 创建一名真人观察者和三名独立 Agent，`agent_coop` 创建四名 Agent。每个 Agent 都有自己的 `AgentBrain`、生成式人物身份、上下文、长期记忆 owner、关系档案、事实图谱、心理矩阵、计划和技能学习系统。

## 私有人物注入

小镇按 `memory_owner_id` 使用独立、稳定的随机种子生成镇民。职业目前包含园艺师、机械师、档案员和旅行者；每个人都有不同的背景、愿望、创伤、价值权重、语言风格、家庭人物、三段形成性记忆和两段日常生活回忆。它们会进入该 Agent 的私有事实图谱与分类长期记忆，不作为公开 MOD 状态广播。玩家最初只能看见表层社交风格，身世仍遵循 `story_reveal` 的事实依据和节奏逐步揭露。

这不是工作→动作规则表。人物材料只为评价、记忆召回、动机、承诺、表达和事实约束提供先验，动作仍从当轮合法动作中动态决策。API 可用 `agent_memory_owner_ids` 为未选择人物卡的 Agent 显式提供稳定 owner；同一个 owner 在不同种子对局中生成相同人物，并可在 SQLite 模式延续长期经历。

## 三日循环

小镇从 07:00 运行到 23:00，每个行动推进 30 分钟，三天后生成终局评价。玩家每行动一次，三个 Agent 会依次行动一次，因此“等待 30 分钟”相当于推进一轮完整镇民日程。

地点包括晨露农场、齿轮工坊、风铃广场、月桂图书馆、镜湖和橡果旅店。合法动作包括：

- `move_to`：沿带 route ID 的路线前往另一地点；
- `work`：在所属工作地点使用对应工具完成日常任务；
- `socialize`：只与同地点的具体对象交谈并更新双向关系；
- `rest`：恢复能量，在旅店效果更好；
- `explore`：在广场或湖边发现环境细节；
- `respond_incident`：在已知事件现场投入精力处理灾害或意外；
- `seek_shelter`：面对已知高风险事件前往旅店避险；
- `check_bulletin`：在广场、图书馆或旅店核对可发现的新闻与公告；
- `support_neighbor`：在压力环境中照顾同地点、状态低落的邻居；
- `check_phone / publish_post`：浏览平台或引用已知事件发布内容；
- `reshare_post / comment_post`：保留来源链转发、质疑或讨论帖子；
- `verify_claim`：用已知记录或图书馆检索结果求证声明；
- `wait`：观察环境并让时间推进。

Agent 的基线策略提供工作、午间社交、傍晚活动和回家休息的日程建议，但最终动作仍由人物性格、心理状态、记忆、关系、有限理性噪声、当前合法动作和可选真实 LLM 共同决定。日程不是固定行为树，Agent 可能改变计划、提前休息或去别处。

## 权威世界模型与传播

`TownEventDirector` 同时管理时钟、季节、天气、温度、风、预报、风险、新闻、公告、政策和活跃事故。镇长讲话、节水政策、区域新闻、工坊事故、暴雨预警、洪水、避险公告、互助政策、停电、辟谣新闻和灾后公共会议不再按固定时间表发放，而由风险事实、前序事件、最小因果延迟、事故状态、发生概率和冷却共同触发。Agent 的维护、救灾和求证行为会反向改变风险事实。事件以带来源、模态、强度、效价、紧迫度、不确定性和权威状态的刺激进入现有多模态管线，因此会影响快速评价、身体反射、心理矩阵、记忆召回和审慎门控。

世界事件会镜像到微博式微帖和短视频平台。Agent 可以用手机浏览、发布有来源的帖子、评论、转发、调查和求证。帖子只代表观察到的报告：转述保留父链并累计失真，未经求证的“水坝已经垮塌”等说法不会进入事实图谱。关系亲近度与私人来源信任会改变每个 Agent 的 feed；重复转发不会被误认为独立信源。完整兼容契约见 [社交媒体兼容层](SOCIAL_MEDIA.md)。

信息不是全知广播：

- `broadcast` 和可直接感知的 `environment` 进入所有居民的观察；
- `local` 只进入事件发生时位于受影响地点的 Agent；后来进入现场且事故仍未解决时，会产生仅对该 Agent 可见的直接观察事件；
- `public_bulletin` 先被公告栏附近的居民获知，其他 Agent 必须执行查阅动作后才能补齐；
- `public_state(viewer_id)` 会按该 Agent 的知识集合过滤新闻、公告、政策和事故；
- 世界事件不通过协作黑板二次泄露；流言只能经显式转述、手机 feed 和求证动作传播，不做隐式共享。

Agent 的环境世界模型只包含自己已观察或收到的事件，并显式标记 `knowledge_scope=observed_local_or_received_broadcast_not_omniscient`。已知事件会生成救灾、避险、查证和互助可供动作；未知事故不会偷偷出现在合法动作列表中。

## 技能实验

MOD 覆盖了两个主要技能族：

- `town_navigation`：场景包含起点、终点、route ID、天气和地图版本；
- `town_work:<job>`：场景包含岗位、任务、工具和工作地点。

同一工作稳定执行五次后可进入自动化；新岗位或新工具仍需要指导。路线熟练度按路线和天气隔离，例如晴天熟悉“旅店→农场”不会让首次雨天路线无条件接管。连续失败会触发 `degraded`，高层重新介入。

社交技能还包含具体互动对象，避免“熟悉与甲相处”被错误迁移成“完全了解乙”。关系分数属于 MOD 公开状态；对对方的私有印象、敏感点和记忆仍留在各自 Agent 的隔离上下文中。

## Godot 运行

先启动后端：

```bash
uvicorn hva_engine.api:app --reload
```

然后用 Godot 4 打开 `godot/project.godot`。场景提供：

- 原生绘制的温暖田园像素地图，不依赖外部版权素材；
- Agent 平滑移动、地点和当前活动气泡；
- 单步与自动观察；
- 每名 Agent 的公开性格表现、能量、心情、工作/社交/公民行动经验、关系、故事揭露和公开技能阶段；
- 最近的移动、工作、公开对话、世界公告和危机响应事件；
- 手机帖子、转发失真、评论和求证结果；
- 暴雨遮罩与雨线、事故地点警戒圈、顶栏世界风险。

Godot 使用公开 `MatchView`，不会读取私有决策事件、提示词、记忆正文或心理推理。`agent_summaries.skill_learning` 只暴露技能数量、阶段计数和已自动化技能名，用作游戏内可观察能力面板。

## 真实 LLM

在环境中把小镇加入 Provider 范围：

```bash
export HVA_AGENT_RUNTIME=llm
export HVA_LLM_MODS=agent_town
```

真实 Provider 仍为三名 Agent 分别构建独立上下文。测试要求 `owner_agent_id` 与事件 actor 一致，并确保私有记忆、关键事实图谱、技能状态、决策倾向、当前观察和合法动作不发生关键截断。进入自动化后，只有当前本地策略推荐且当前场景熟悉的技能可以绕过 Provider。

## 当前边界

- 视觉移动是领域动作之间的插值，还不是连续寻路、局部避障或碰撞执行器。
- 对话在基线模式使用人物语言风格实现器；启用真实 LLM 后才生成开放式语义表达。
- 当前关系是双向标量加私有关系/来源信任记忆；社交媒体流言已有来源链、独立信源识别和主动调查，但尚未实现面对面口耳传播、邀约、冲突修复和共同事件记忆。
- SQLite 已能恢复权威世界，但长期常驻服务器仍需要离线时间推进、快照压缩、迁移和多进程写入仲裁。
- 当前因果规则仍使用人工声明的事件模板；后续需要加入更长周期经济、人口、资源和机构模型，并用消融验证每条因果边是否真正改变行为。
