# 评价体系（MVP-11）

评价不是发布后的报表，而是引擎协议的一部分。每个合法动作、领域事件和终局结果都进入统一事件流，使不同 MOD 可以用同一量纲比较。

## 三层证据与六维代理指标

MVP-11 不再用一个综合分暗示“已经像真人”。每局同时报告：

- `structural_integrity`：规则遵守、viewer-scoped 观察隔离、上下文归属和事实图谱接地；
- `behavioral_quality`：人类感行为代理、任务智能与局内动态性；
- `player_experience`：只能来自真人盲评或真实遥测；未采集时固定为 `null`。

兼容字段 `composite_score` 仍保留用于纵向研究基准，但明确标记为 `research_proxy_not_human_validation`。

| 维度 | MVP 可观测量 | 权重 | 目标 |
|---|---|---:|---:|
| 玩家参与度 | 人类行动占比、行动多样性（真实遥测接入前折扣） | 按模式 | 0.60 |
| 引擎通用性 | MOD 契约覆盖、能力标签覆盖 | 按模式 | 0.80 |
| 动态性 | 分数轨迹变化、方向反转、动作与事件多样性 | 按模式 | 0.65 |
| 虚拟玩家评价 | 对抗公平性或协作任务表现 | 按模式 | 0.60 |
| AI 对手智能性 | 预测验证、记忆影响、策略多样性、任务表现 | 按模式 | 0.65 |
| AI 人类感 | 人格/身份、显著性记忆、证据反思、评价情绪、社会信念、计划、有限理性、动机冲突、后果迟滞、身份失调、诱惑/社会压力、承诺冲突、延迟后果、表达差值与故事揭露 | 按模式 | 0.72 |

综合分使用模式权重；Agent-only 对局不包含玩家参与度，也不会填充中性假分。唯一运行时配置位于 `src/hva_engine/data/evaluation_v11.json`；每份报告保存版本和配置 SHA-256，避免代码、文档和报表使用不同权重。

所有分数都归一化到 `[0, 1]`。当前指标刻意保持简单、可解释，等收集真实样本后再做校准，避免在没有基线时制造“精确幻觉”。

AI 对手智能性同时提供不折叠的 `ai_capability_profile`：

- `rules_compliance`：决策通过合法动作硬约束的比例
- `world_model_grounding`：决策前是否形成基于当前观察的世界模型
- `prediction_accuracy`：预期规则事件是否实际发生
- `memory_utilization`：决策是否检索了历史经验
- `memory_influence_rate`：记忆是否真正改变了基线选择
- `decision_planning`：是否记录了机器可验证的预期效果
- `adversarial_competitiveness`：对抗模式中的实力接近程度
- `cooperation_quality`：协作模式中的协调行为与共享结果
- `human_likeness`：人类感综合代理分
- `human_likeness_components`：除人格/身份、心理、对手、意图、有限理性和叙事外，还包含记忆检索依据、反思证据、评价—情绪连贯性、结果再评价、情绪方向性、社会信念、情境人格激活、计划持续/合理重规划、内在情绪—公开表达差值，以及动机冲突、后果迟滞、身份失调、诱惑/社会压力、承诺冲突和延迟后果实现
- `character_card_grounding`：使用人物卡时，检查每次决策的稳定人格与身份层是否仍指向所选卡，并明确报告 `scripted_action_pool=false`
- `strategic_influence`：报告私有意图覆盖/隔离、目标一致性、误导/利诱/威慑尝试率、边界遵守、事实防火墙、连续向量多样性和非负结果代理率
- `implicit_control`：除刺激、反射和 Provider 跳过率外，报告技能诊断覆盖、指导练习、本地自动接管、新场景重新求助及 `novel/guided/practiced/automatic/degraded` 阶段迁移

`adversarial_interview` 额外返回 `mod_specific_profile`，包括：

- 问题主题覆盖率
- 心理反应幅度（过于平坦和纯随机剧烈波动都会扣分）
- MOD 压力信号是否进入心理矩阵
- 身份解释与自传记忆使用
- 故事揭露的事实图谱溯源
- 回答策略多样性
- 单轮策略混合复杂度：单一模板和无差别堆叠都会扣分，适度冲突与组合得分更高
- 人物弧光阶段变化和终局完整度
- 镇定、真实性、一致性与信任构成的韧性分
- 文本行为分：精确非重复、字符 n-gram 近重复、问题贴合、实质表达和 Provider 自主生成率

`agent_town` 的 `mod_specific_profile` 不评价台词好坏，而检查世界—人物闭环：事件家族覆盖、因果边可追踪、按观察者隔离的本地传播、Agent 世界模型接地、生成身份可辨识度、事故响应/解决率、避险/查证/互助等反应多样性、心理矩阵变化、手机行为多样性、转述溯源保留和流言纠正率。该分仍是引擎代理指标，不替代玩家对“反应是否自然、故事是否有趣”的盲评。真人证据通过独立的 [真实 LLM 与真人自然度盲评](BLIND_NATURALNESS_EVAL.md) 收集，不与代理分混写。

规则遵守是准入门槛而非加分项；出现非法 Agent 决策时，`valid_for_comparison=false` 且综合分归零。Agent-only 基线局的玩家参与度为 `null`，不会参与加权。

MVP-11 自动分只证明事件流中出现了所要求的结构和行为表征，不证明 Agent 已等同真人。文本指标能识别逐字重复和高度相似模板，但仍不能独立判断讽刺、文化语境或深层人物忠实度。`provider_execution` 另行报告 Provider/模型、token、延迟、错误与 fallback；只要真实 Provider 测试发生 fallback，`valid_for_provider_comparison=false`。最终校准必须使用真人双盲 A/B。

## 双轨人物评价

`hva-narrative-calibration` 单独报告人类创作虚构人物与机构传记人物的关键抉择、动机排序、心理评价、弧光和置信度拟合。该结果不进入六维对局综合分，也不得标为真人行为数据。内置 38 例只是原型机制回归集，作者设计的标签、权重和 holdout 会造成乐观偏差；传记是选择性叙述，不是行为遥测。详细版权门禁、负面对照和结果见 [叙事人物决策校准](NARRATIVE_CHARACTER_CALIBRATION.md)。

另一轨 `real_human_behavior_fidelity` 必须来自真实玩家遥测、真人脚本或双盲实验，并保留人群、语言、文化、实验条件和置信区间。两轨只有在独立样本上验证相关性后才能讨论合并。

## 镜像与公平性基准

`python -m hva_engine.benchmark` 对每个对抗种子交换双方席位，分别报告固定 Agent 身份、零号席位和行动优先权的胜率等价值。随机分配先手只能消除身份偏差；地图席位与规则先后手优势仍分别由 `seat0_win_equivalent` 和 `initiative_win_equivalent` 检测。协作模式报告成功/失败分布与跨种子综合分标准差。

基准的 `balance` 区块把等价值转换为 0–1 公平分，并单独提供重复平局惩罚；协作模式以 65% 成功率作为默认可玩性中心。它们不会偷偷改写单局综合分，而是作为跨局发布门槛使用。

## 下一阶段的测量升级

### 玩家参与度

- 真实会话留存、退出回合、动作响应时延
- 主动决策占比与“无意义点击”过滤
- 直播投票人数、投票集中度与复投率

### 通用性

- 新 MOD 接入工时与定制代码行数
- 隐藏信息、并行回合、异步行动、多人阵营的契约测试
- Godot 展示组件复用率

### 动态性

- 领先者变化、状态熵、可逆/不可逆转折
- 同种开局的轨迹差异度
- 玩家对“意外但合理”的主观评分

### 虚拟玩家评价

- 角色一致性、可理解性、公平感与娱乐性问卷
- 虚拟观众对战况和行为的实时打分
- Agent 人设在跨 MOD 中的稳定性

### AI 智能性

- Elo/Glicko、相对遗憾值、搜索基线胜率
- 不同水平玩家的目标胜率区间
- 决策耗时、成本和非法动作率

### AI 人类感与叙事

- 真人盲评“是否像稳定角色而非每回合重置的工具”
- 心理状态与实际受挫、压力和惊讶之间的连贯性
- 角色在多个 MOD 中的身份一致性和行为可辨识度
- 故事揭露节奏、玩家主动追问率和关键片段记忆率
- 事实图谱冲突率、自由发挥提案拒绝率与修订链长度
- 承诺债务的形成、修复与延迟后果是否被玩家认为“意外但合理”

## 迭代门槛

每个 MVP 至少运行 100 局基线 Agent 自博弈和 30 局真人对战，再讨论权重变化。架构改造优先级按以下顺序：低于目标的维度、影响的 MOD 数量、修复成本。评价版本必须随事件数据保存，确保纵向结果可比。
