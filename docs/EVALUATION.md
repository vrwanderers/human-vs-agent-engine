# 评价体系（MVP-1）

评价不是发布后的报表，而是引擎协议的一部分。每个合法动作、领域事件和终局结果都进入统一事件流，使不同 MOD 可以用同一量纲比较。

## 五维指标

| 维度 | MVP 可观测量 | 权重 | 目标 |
|---|---|---:|---:|
| 玩家参与度 | 人类行动占比、行动类型多样性 | 28% | 0.68 |
| 引擎通用性 | MOD 契约覆盖、能力标签覆盖 | 20% | 0.80 |
| 动态性 | 领域事件类型、有效状态变化 | 18% | 0.65 |
| 虚拟玩家评价 | Agent 归一化对局表现 | 14% | 0.60 |
| AI 对手智能性 | 合法率、策略多样性、终局竞争性 | 20% | 0.65 |

综合分为加权和：

```text
S = 0.28E + 0.20G + 0.18D + 0.14V + 0.20A
```

所有分数都归一化到 `[0, 1]`。当前指标刻意保持简单、可解释，等收集真实样本后再做校准，避免在没有基线时制造“精确幻觉”。

AI 对手智能性同时提供不折叠的 `ai_capability_profile`：

- `rules_compliance`：决策通过合法动作硬约束的比例
- `world_model_grounding`：决策前是否形成基于当前观察的世界模型
- `memory_utilization`：除首轮外的决策是否检索了历史经验
- `decision_planning`：是否记录可检验的预期效果
- `adversarial_competitiveness`：对抗模式中的实力接近程度
- `cooperation_quality`：协作模式中的协调行为与共享结果

Agent-only 基线局不适用“玩家参与度”，API 会在 `applicability` 标记中明确指出，并以中性值参与 MVP 综合分，避免误读为真人参与度数据。

## 下一阶段的测量升级

### 玩家参与度

- 真实会话留存、退出回合、动作响应时延
- 主动决策占比与“无意义点击”过滤
- 直播投票人数、投票集中度与复投率

### 通用性

- 新 MOD 接入工时与定制代码行数
- 隐藏信息、并行回合、异步行动、多人阵营的契约测试
- Godot 展示组件复用率

### 动态性

- 领先者变化、状态熵、可逆/不可逆转折
- 同种开局的轨迹差异度
- 玩家对“意外但合理”的主观评分

### 虚拟玩家评价

- 角色一致性、可理解性、公平感与娱乐性问卷
- 虚拟观众对战况和行为的实时打分
- Agent 人设在跨 MOD 中的稳定性

### AI 智能性

- Elo/Glicko、相对遗憾值、搜索基线胜率
- 不同水平玩家的目标胜率区间
- 决策耗时、成本和非法动作率

## 迭代门槛

每个 MVP 至少运行 100 局基线 Agent 自博弈和 30 局真人对战，再讨论权重变化。架构改造优先级按以下顺序：低于目标的维度、影响的 MOD 数量、修复成本。评价版本必须随事件数据保存，确保纵向结果可比。
